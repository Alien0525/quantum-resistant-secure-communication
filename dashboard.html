Key Insights & Analysis
Kyber768 Dominates on Speed
Kyber key generation is ~70,000× faster than McEliece. Sub-millisecond operations make it ideal for ephemeral key exchange in TLS-like protocols requiring a fresh keypair per session.
McEliece Public Key Problem
McEliece's ~1 MB public key is ~885× larger than Kyber's — impractical for IoT or mobile. However, its KEM ciphertext is ~6× smaller, an advantage once keys are distributed.
Both Are Equally Quantum-Resistant
McEliece offers NIST Level 5 (~2²⁵⁶), Kyber Level 3 (~2¹⁹²). Neither Module-LWE nor Syndrome Decoding has a known quantum speedup. Shor's and Grover's algorithms offer no advantage against either scheme.
Maturity & Standardization
McEliece (1978, 46 years) has the longest cryptanalysis record. Kyber was standardized as FIPS 203 / ML-KEM in 2024, giving it the strongest regulatory backing. Both are NIST-endorsed for new deployments.
Key Length Justification
Kyber768: k=3 module rank achieves NIST Level 3 with balanced size/speed. McEliece-6960119: (n=6960, t=119) targets Level 5 — appropriate for highest-assurance, long-lived key scenarios.
Hybrid KEM + AES-256-GCM Design
Both schemes use KEM to derive a 32-byte shared secret, then HKDF-SHA256 to derive an AES-256 key. AES-256-GCM then provides confidentiality, 128-bit integrity tags, and replay protection via nonces + timestamp AAD.
Practical Use Case Split
Kyber768: TLS, messaging, API auth, IoT, mobile — any latency-sensitive protocol. McEliece: Top-secret classified data, long-lived key pairs distributed once, ultra-conservative deployments.
"Harvest Now, Decrypt Later" Threat
Nation-state adversaries collect today's RSA/ECDH-encrypted traffic to decrypt once a CRQC arrives. Data encrypted via Kyber/McEliece + AES-256-GCM today remains secure even after a CRQC exists.
Conclusions
1
Shor's Algorithm Breaks All of NSA Suite B
A "very powerful" quantum computer — defined as ~4,096 logical qubits (~20 million physical qubits with surface-code error correction) — running Shor's algorithm breaks RSA-2048 and all elliptic-curve schemes in polynomial time O(n³). Current estimates place such a machine 10–20 years away. Both Kyber768 and Classic McEliece are immune because their hard problems (Module-LWE, Syndrome Decoding) have no known quantum polynomial algorithm.
2
Grover's Algorithm Halves Symmetric Security — AES-256 Survives
Grover's quadratic speedup reduces AES-128 to 64-bit effective security (insufficient post-quantum). AES-256 retains 128-bit quantum security — the accepted safe threshold. Our implementation uses AES-256-GCM throughout, ensuring no vulnerability. The mitigation is simply doubling symmetric key sizes, as NSA recommends.
3
Kyber768 is the Practical Winner for Most Applications
With ~70,000× faster key generation, 885× smaller public keys, and sub-millisecond total operation times, Kyber768 (FIPS 203 / ML-KEM, 2024) is the right choice for TLS replacement, encrypted messaging, and API security. Its NIST Level 3 security far exceeds any foreseeable classical threat.
4
McEliece is the Ultra-Secure Conservative Option
Classic McEliece's 46-year cryptanalysis record and NIST Level 5 security make it the right choice for classified government systems, critical infrastructure, and long-term data archival. Its 194-byte ciphertext is the smallest among all NIST PQC finalists, making encapsulation/decapsulation efficient once the one-time key setup cost is amortized.
5
Hybrid KEM + AES-256-GCM Defeats All Three Required Attacks
The secure communication scheme from class remains structurally valid — only the public-key component is replaced. Eavesdropping: defeated by Module-LWE / Syndrome Decoding hardness. Modification: detected by AES-256-GCM 128-bit auth tag — any bit flip causes verifiable failure. Replay: blocked by per-message nonce tracking and timestamp AAD validation. The combination provides comprehensive post-quantum security with negligible overhead.
